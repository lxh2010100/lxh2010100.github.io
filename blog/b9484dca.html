Ubuntu22.04安装jenkins

Ubuntu jenkins

Jenkins 是一个支持持续集成 （CI） 和持续交付 （CD） 的 CI/CD 软件平台。它用于自动化软件测试、构建、交付和部署。在 Ubuntu 上使用 Jenkins 创建了一个强大的管理工具，将开发工作流程提升到一个新的水平。

本指南将帮助您在 Ubuntu 22.04 系统上安装 Jenkins。

先决条件

  • 运行 Ubuntu 22.04 Jammy Jellyfish 的系统。
  • 具有管理员权限的用户帐户。
  • 访问终端窗口/命令行（CTRL+ALT+T）。
  • Java 8 或 11 安装在 Ubuntu 上。

1 安装 Java

Jenkins 需要 Java 运行时环境 （JRE）。本指南将 OpenJDK 用于 Java 环境。OpenJDK是一个开发工具包，包括Java运行时环境。

在撰写本文时，Jenkins 仅支持 Ubuntu 上的 Java 8 和 Java 11。您可以在 Ubuntu 上安装多个版本的 Java。如果这样做，请确保将 Java 8 或 Java 11 设置为默认版本。

请按照以下步骤在 Ubuntu 上安装 Java：

  1. 检查您的 Ubuntu 系统上是否已经安装了 Java：
java -version

由于我们的系统上没有安装 Java，我们将使用 OpenJDK 安装它。

注意：如果您的 Ubuntu 系统上安装了 Java，请跳到步骤 2。

  1. 首先，打开终端窗口并通过运行以下命令更新系统包存储库：
sudo apt update

输入您的管理员密码并等待更新完成。

  1. 根据要安装的 Java 版本（Java 8 或 11），运行以下命令之一：
  • 要安装 OpenJDK 8，请运行：
sudo apt install openjdk-8-jdk -y
  • 要安装OpenJDK 11，请运行：
sudo apt install openjdk-11-jdk -y

我们在系统上安装了OpenJDK版本11：

注意：阅读我们的 Jenkins 教程，熟悉 Jenkins 并开始熟练地使用它。

2 添加 Jenkins 存储库

建议使用项目维护的存储库安装 Jenkins，而不是从默认的 Ubuntu 存储库安装。原因是因为默认 Ubuntu 存储库中的 Jenkins 版本可能不是最新的可用版本，这意味着它可能缺少最新的功能和错误修复。

按照以下步骤将 Jenkins 存储库添加到您的 Ubuntu 系统中。

  1. 首先导入 GPG 密钥。GPG 密钥验证包完整性，但没有输出。跑：
curl -fsSL https://pkg.jenkins.io/debian-stable/jenkins.io.key | sudo tee /usr/share/keyrings/jenkins-keyring.asc > /dev/null
  1. 将 Jenkins 软件仓库添加到源列表中，并提供认证密钥：
echo deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc] https://pkg.jenkins.io/debian-stable binary/ | sudo tee /etc/apt/sources.list.d/jenkins.list > /dev/null

该命令将长期支持 （LTS） 稳定版本添加到源列表中，但没有输出。

3 安装 Jenkins

参考文章：https://blog.csdn.net/leeasj/article/details/129990269

curl -fsSL https://pkg.jenkins.io/debian-stable/jenkins.io-2023.key | sudo tee \
  /usr/share/keyrings/jenkins-keyring.asc > /dev/null
echo deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc] \
  https://pkg.jenkins.io/debian-stable binary/ | sudo tee \
  /etc/apt/sources.list.d/jenkins.list > /dev/null
sudo apt-get update
sudo apt-get install jenkins

设置先决条件后，请按照以下步骤在 Ubuntu 上安装 Jenkins：

  1. 再次更新系统存储库。更新会刷新缓存，并使系统知道新的 Jenkins 存储库。
sudo apt update
  1. 通过运行以下命令安装 Jenkins：
sudo apt install jenkins -y

等待下载和安装完成。

  1. 要检查 Jenkins 是否已安装并正在运行，请运行以下命令：
sudo systemctl status jenkins

输出中应显示标记为**活动（正在运行）**的亮绿色条目，指示服务正在运行。

  1. 按 Ctrl+Z 退出状态屏幕。

注意：如果 Jenkins 服务未运行或未处于活动状态，请运行以下命令以启动它：

sudo systemctl enable --now jenkins

如果systemcrl ，可以尝试下列命令

sudo service jenkins enable
sudo service jenkins start

4 修改防火墙以允许 Jenkins

允许 Jenkins 通过设置默认的 UFW 防火墙进行通信。

  1. 通过运行以下命令打开端口 8080：
sudo ufw allow 8080
sudo ufw allow OpenSSH
sudo ufw status

如果您使用的是其他防火墙应用程序，请按照其特定说明允许端口 8080 上的流量。

  1. 如果尚未配置 UFW 防火墙，它将显示为非活动状态。通过运行以下命令启用 UFW：
sudo ufw enable

sudo ufw status

5 设置 Jenkins

请按照以下步骤设置 Jenkins 并开始使用它：

  1. 打开 Web 浏览器，然后导航到服务器的 IP 地址。使用以下语法：
http://ip_address_or_domain:8080

使用您使用 Jenkins 的服务器的实际 IP 地址或域名。例如，如果您在本地运行 Jenkins，请使用

http://localhost:8080

将打开一个页面，提示您解锁 Jenkins。在下一步中获取所需的管理员密码。

  1. 通过打开终端并运行以下命令来获取默认的 Jenkins 解锁密码：
sudo cat /var/lib/jenkins/secrets/initialAdminPassword

  1. 系统返回一个字母数字代码。在管理员密码字段中输入该代码，然后单击继续。

  2. 安装程序会提示安装建议的插件或选择要安装的插件。只需安装建议的插件即可。

  1. 您以后总是可以安装更多插件。系统继续进行Jenkins的初始设置。

    下一步是创建第一个管理员用户。输入您要用于Jenkins管理员的凭据，然后单击保存并继续。

  1. 在此之后，您应该设置实例配置。这是此 Jenkins 安装的首选网络地址。确认要用于服务器的地址。这很可能与您用于访问此配置页面的地址相同。

指定 Jenkins URL 后，单击“保存并完成”。

  1. 你应该看到一个页面，上面写着 Jenkins 已经准备好了！单击“开始使用 Jenkins”以打开 Jenkins 仪表板。

注意：查看我们关于如何手动重新启动 Jenkins 的指南 - 在解决问题时很有用的做法。
